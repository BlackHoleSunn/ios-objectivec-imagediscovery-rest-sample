    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1><a id="user-content-office-365-microsoft-graph-image-discovery-for-ios" class="anchor" href="#office-365-microsoft-graph-image-discovery-for-ios" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" role="img" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Office 365 Microsoft Graph Image Discovery for iOS</h1>

<p>Image Discovery is an iOS sample app that allows you either to take a picture or to search the web for one, and process the fetched image. You can then save it in the cloud or mail to a recipient. For example, you can take a picture, convert the image to black and white, and push it to OneDrive for Business to store.</p>

<p><a href="https://github.com/OfficeDev/O365-iOS-Microsoft-Graph-Image-Discovery/blob/master/Images/camera.gif" target="_blank"><img src="https://github.com/OfficeDev/O365-iOS-Microsoft-Graph-Image-Discovery/raw/master/Images/camera.gif" alt="Camera" style="max-width:100%;"></a></p>

<p>Or, you can search for an image via Google Custom Search and mail it to a friend.</p>

<p><a href="https://github.com/OfficeDev/O365-iOS-Microsoft-Graph-Image-Discovery/blob/master/Images/search.gif" target="_blank"><img src="https://github.com/OfficeDev/O365-iOS-Microsoft-Graph-Image-Discovery/raw/master/Images/search.gif" alt="Search" style="max-width:100%;"></a></p>

<p>This sample illustrates how to use Microsoft Graph, a unified API endpoint, for working with mail and files in Office 365. For more information about Microsoft Graph see the <a href="https://graph.microsoft.io">Microsoft Graph overview page</a>. </p>

<h2><a id="user-content-prerequisites" class="anchor" href="#prerequisites" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" role="img" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Prerequisites</h2>

<ul>
<li><a href="https://developer.apple.com/xcode/downloads/">Xcode</a> from Apple</li>
<li>Installation of <a href="https://guides.cocoapods.org/using/using-cocoapods.html">CocoaPods</a>  as a dependency manager.</li>
<li><p>An Office 365 account. You can sign up for <a href="https://portal.office.com/Signup/Signup.aspx?OfferId=6881A1CB-F4EB-4db3-9F18-388898DAF510&amp;DL=DEVELOPERPACK&amp;ali=1#0">an Office 365 Developer subscription</a> that includes the resources that you need to start building Office 365 apps.</p>

<blockquote>
<p>Note: If you already have a subscription, the previous link sends you to a page with the message <em>Sorry, you canât add that to your current account</em>. In that case, use an account from your current Office 365 subscription.</p>
</blockquote></li>
<li><p>A Microsoft Azure Tenant to register your application. Azure Active Directory (AD) provides identity services that applications use for authentication and authorization. A trial subscription can be acquired here: <a href="https://account.windowsazure.com/SignUp">Microsoft Azure</a>.</p>

<blockquote>
<p>Important: You'll also need to ensure your Azure subscription is bound to your Office 365 tenant. To do this, see the Active Directory Team Blog post, <a href="http://blogs.technet.com/b/ad/archive/2013/11/08/creating-and-managing-multiple-windows-azure-active-directories.aspx">Creating and Managing Multiple Windows Azure Active Directories</a>. The section <strong>Adding a new directory</strong> will explain how to do this. You can also see <a href="https://msdn.microsoft.com/office/office365/howto/setup-development-environment#bk_CreateAzureSubscription">Set up your Office 365 development environment</a> and the section <strong>Associate your Office 365 account with Azure AD to create and manage apps</strong> for more information.</p>
</blockquote></li>
<li><p>A Client Id and Redirect Uri values of an application registered in Azure. This sample application must be granted the <strong>Have full access to user files and files shared with user</strong> permission and <strong>Send mail as user</strong> permission for <strong>Microsoft Graph</strong>. To create the registration, see <strong>Register your native app with the Azure Management Portal</strong> in <a href="https://msdn.microsoft.com/en-us/office/office365/howto/add-common-consent-manually">Manually register your app with Azure AD so it can access Office 365 APIs</a> and <a href="https://github.com/OfficeDev/O365-iOS-Microsoft-Graph-Image-Discovery/wiki/Grant-permissions-to-the-Connect-application-in-Azure">grant proper permissions</a> in the sample wiki to apply the proper permissions to it.</p></li>
<li>This sample uses <strong>Google Custom Search</strong> to search for images. In order for the search functionality in this app to work, you'll need your own API key and custom search engine ID (cx). For more information about obtaining these values see <a href="https://developers.google.com/custom-search/docs/overview">Google Custom Search</a>.
&gt; Note: The search functionality in this app is reliant on getting the API Key and search engine ID. The camera workflow, for getting an image, will still function without this information, but again you'll need these values for the search component in the app to work. Also, Google Custom Search is just being used as an example; you can implement other search engine options as needed.</li>
</ul>

<h2><a id="user-content-running-this-sample-in-xcode" class="anchor" href="#running-this-sample-in-xcode" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" role="img" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Running this sample in Xcode</h2>

<ol>
<li>Clone this repository.</li>
<li><p>Use CocoaPods to import these dependencies:</p>

<pre><code> pod 'AFNetworking', ' ~&gt; 3.0'
 pod 'ADALiOS'
</code></pre>

<p>This sample app already contains a podfile that will get the ADAL and AFNetworking components (pods) into  the project. Simply navigate to the project from <strong>Terminal</strong> and run: </p>

<pre><code>   pod install
</code></pre>

<p>For more information, see <strong>Using CocoaPods</strong> in <a href="#AdditionalResources">Additional Resources</a>.</p></li>
<li><p>Open <strong>ImageDiscovery.xcworkspace</strong>.</p></li>
<li><p>Open <strong>AuthenticationConstants.m</strong> under <strong>Library/Authentication</strong>. You'll see that the <strong>ClientID</strong> and <strong>RedirectUri</strong> values can be added to the top of the file. Also you'll need to provide the Google API Key and the custom search engine ID (cx) for the search functionality to work. Supply the necessary values here:</p>

<pre><code>// You will set your application's Client ID and Redirect URI.
NSString * const kRedirectUri = @"ENTER_YOUR_REDIRECT_URI";
NSString * const kClientId    = @"ENTER_YOUR_CLIENT_ID";
NSString * const kResourceId  = @"https://graph.microsoft.com";
NSString * const kAuthority   = @"https://login.microsoftonline.com/common";

//To enable image search in the app, you'll need to supply the API Key and custom search engine ID (cx) after registering for Google Images.
NSString * const kGoogleAPIKey = @"ENTER_Google_API_KEY";
NSString * const kGoogleCX     = @"ENTER_Google_CX";
</code></pre>

<blockquote>
<p>Note: If you have don't have CLIENT_ID and REDIRECT_URI values, <a href="https://msdn.microsoft.com/library/azure/dn132599.aspx#BKMK_Adding">add a native client application in Azure</a> and take note of the CLIENT_ID and REDIRECT_URI values.</p>
</blockquote></li>
<li><p>Run the sample.</p></li>
</ol>
